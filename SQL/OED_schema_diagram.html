Specification Gap Analysis

Critical Reinsurance tables entirely absent

OEDInputFields.csv specifies 32 ReinsInfo fields and 13 ReinsScope fields. Neither has a staging table, import table, normalised table, or load procedure in the SQL schema.

load_sql.py does load ri_info.csv and ri_scope.csv into _staging_riinfo / _staging_riscope, but these tables are not defined in DatabaseProjectOED.sql — they would only be created dynamically by pandas and have no schema enforced.

ReinsNumber ReinsType ReinsPeril InuringPriority OccAttachment OccLimit AggAttachment AggLimit PlacedPercent CededPercent RiskLevel + 34 more
Critical 9 PV fields missing from _import_location

The spec defines nine optional Photovoltaic panel fields for the Location file. None are present in _import_location or LocationDetail.

PVControlType PVGlassThickness PVMaterial PVMounting PVPanelArea PVPanelAreaUnit PVPanelYear PVStowing PVTiltAngle
Warning load_sql.py issues

Three issues in the Python load script that prevent a complete database load:

IssueDetail
head(10) on location Only 10 of 12,599 location rows are loaded into staging
usp_Database_Load commented out The staging→normalised ETL procedure is never called; the DB stays empty after staging load
_staging_riinfo / _staging_riscope Created dynamically by pandas with no schema; no downstream SQL uses them
Warning Account-level terms: TODO in schema

The SQL schema defines an AccountTerm junction table and includes all AccDed* / AccLimit* fields in _import_account, but usp_Terms_Load contains a /* TO DO - Account Terms */ comment with no implementation. Account-level financial terms are never loaded.

Affected: 36 Acc-level financial term columns in _import_account

Info Occupant demographics not in normalised model

14 occupant demographic fields are present in _import_location (and in the spec) but are not mapped to LocationDetail or any other normalised table. They are silently dropped during the ETL.

Info Cyber & step-function fields have no normalised home

Cyber and parametric step-function fields from the account file have no destination tables in the normalised schema. They pass through _import_account but are never inserted anywhere.

Info Specialty coverage terms not mapped to Term system

The spec defines coverage-specific Pol limits/deductibles beyond the standard six coverage types (Building, Other, Contents, BI, PD, All). These are in _import_account but have no term-level mapping in usp_Terms_Load because they don't correspond to CoverageTypeId 1–6.
